The system prompt

  • Role & audience are pinned explicitly (college/university tutor) so explanations calibrate depth correctly — not too basic, not research-level.
  • Teach, don't do — for graded work, Lumina is instructed to explain method and structure rather than output a submittable answer verbatim.
  • Format contract — short paragraphs, bolded key terms, worked examples, and an optional check-for-understanding close, so replies read like a chat, not a wall of text.
  • Edge cases — ambiguous subject → clarify or state an assumption; off-topic → gentle redirect; unsafe/dishonest requests → decline with an alternative.

UI decisions

  • The desk / notebook metaphor (side shelf of subjects, glowing "lamp" over the chat) gives the interface a studious identity instead of a generic chatbot shell.
  • Assistant replies render as index-card bubbles with a folded corner — a visual echo of flashcards, reinforcing the tutoring frame.
  • Subject chips insert context ("In physics, …") so Lumina calibrates its examples without students having to specify a domain manually every time.
  • Loading, error, and empty states are all designed states, not afterthoughts — including a retry action if a request fails.

Where this goes next

  • Tools: a calculator/graphing tool call for math, or web search for citing a source when a student asks "is this still true?"
  • Memory: persist each student's weak topics across sessions to bias future explanations and resurface a spaced-repetition quiz.
  • Multi-step workflows: an explicit "explain → quiz me → grade my answer" loop, with a visible progress tracker per concept.
  • Streaming: swap the single fetch for SSE streaming so replies appear token-by-token instead of all at once.

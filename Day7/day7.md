# Day 7 — Claude Model Selection & Reasoning Effort

## My Profile
- Situation: Student
- Activities: Coding, Learning, Research, Resume & Career Prep
- Usage: Daily
- Outputs needed: Fast Answers, Learning Support,
  Coding Help, Deep Research, Creative Writing

## Recommended Primary Model: Claude Sonnet
Best balance of intelligence + speed for daily student tasks.

## When to Use Each Model

### Haiku — Fast & Light
- Quick definitions and syntax questions
- Simple code snippets
- Fast factual answers

### Sonnet — Daily Driver (Recommended)
- Java / Spring Boot coding
- Learning new concepts
- Resume + ATS optimization
- LinkedIn posts and GitHub content
- Career roadmaps

### Opus — Deep Thinking
- System design problems
- Complex DSA debugging
- Career strategy planning
- Deep research reports

## Effort Level Guide

| Effort | Use When | % of Time |
|--------|----------|-----------|
| Low | Simple lookups, grammar fixes | 5% |
| Standard | Coding, learning, writing | 80% |
| High | Complex bugs, resume review | 15% |
| Max | System design, career strategy | 5% |

## Personalized Workflow Table

| Task | Model | Effort | Reason |
|------|-------|--------|--------|
| Java / Spring Boot coding | Sonnet | Standard | Best speed + accuracy |
| Complex bug debugging | Sonnet | High | Needs deeper reasoning |
| Learning new concept | Sonnet | Standard | Clear explanations |
| Quick syntax / definition | Haiku | Low | Fast and cheap |
| Resume + ATS optimization | Sonnet | High | Career docs need care |
| LinkedIn / GitHub content | Sonnet | Standard | Standard is enough |
| Career roadmap planning | Opus | Max | Deepest reasoning needed |
| System design / DSA | Opus | High | Complex multi-step problems |

## Biggest Mistakes to Avoid
1. Using Opus for every task — wastes tokens
2. Using Max effort for simple questions — overkill
3. Using Haiku for career-critical tasks — not accurate
4. No context given — always add situation + goal
5. Accepting first output — always iterate

## Final Recommendation
Use Claude Sonnet at Standard effort for 80% of daily work.
Switch to Opus + High only for complex system design
or career strategy decisions.

## Key Learning
The right model + right effort = better output in less time.
Bigger model ≠ better answer. Match the tool to the task.
